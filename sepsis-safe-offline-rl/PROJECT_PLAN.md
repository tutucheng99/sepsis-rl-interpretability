# 研究执行计划：ICU脓毒症安全离线强化学习策略开发与审计

## 项目概述

本文件旨在将提供的《离线条件下ICU脓毒症动态治疗安全RL方案》转化为详细的、可执行的分步研究计划。研究团队应严格按照此计划执行，并特别注意每个阶段的交付物、注意事项和检查点，以确保项目的安全性和有效性。

---

## 0. 预备阶段：基础设施与数据准备

**周期：** 1-2 周
**目标：** 建立统一、可复现的实验环境与数据基线。

### 执行步骤

#### 1. 环境与工具链配置
- 配置 `Gym-Sepsis` 环境，确认状态空间（S，含SOFA核心特征）和动作空间（A，离散 5x5 网格）
- 统一计算环境（固定 Python、d3rlpy、PyTorch/TensorFlow 版本）
- **数值稳定与工程一致性（参考3.5）：** 全局设定统一精度（FP32）
- 建立Git版本控制和实验跟踪系统（如MLflow）

#### 2. 数据划分与隔离
- 抽取约 10k 条患者轨迹
- 严格执行划分：训练集（约 9k）、验证集（约 0.5k）、测试集（约 0.5k）
- **铁律：** 测试集在此阶段后必须封存，直到 Phase 4 主评估阶段
- 禁止用于任何调参或设阈值（参考二）

#### 3. 特征工程与预处理标准化
- 定义并锁定特征归一化和缺失值插补策略
- 在训练集上拟合预处理器，并应用于所有数据集
- 确保训练与推断的预处理逻辑绝对一致并锁定版本

### 交付物
- 标准化的数据集
- 锁定的预处理脚本
- 配置好的实验环境

## Phase -1：数据层因果–公平审计与可用性闸门

**周期：** 3-4 周
**目标：** 在训练前评估数据适用性，识别偏见与混淆风险，划定 RL 的有效边界（参考 3.1）

### Task 1.1: 行为策略估计与重叠度分析

1. **拟合行为策略：** 使用训练集拟合临床医生的行为策略 $\pi_{\text{beh}}(a|s,t)$
2. **重叠度评估：** 计算倾向评分（Propensity Scores），评估状态-动作对的重叠度（Overlap/Positivity）
3. 识别数据支持不足的区域

### Task 1.2: 因果图构建与混淆敏感性分析

1. **构建最小因果图 (DAG)：** 明确 S, A, R, T 关系
2. 将"测量频度"或"缺失指示"作为医生行为的影子变量纳入 DAG，识别潜在混淆因子
3. **隐性混淆敏感性分析：** 采用 Rosenbaum 界或 E-value 方法，参数化评估"推翻结论所需混淆强度"
4. **不确定性量化：** 将结果映射为德尔菲不确定性 $U_{\text{del}}(s,a)$，用于后续悲观化评价

### Task 1.3: 公平性审计与可拼接性闸门

1. **代表性与偏见审计：** 按关键群体（如年龄、性别、基线 SOFA 评分）分层报告样本覆盖率和行为差异
2. **可拼接性（Stitchability）评估：** 评估跨不同行为模式（聚类）的状态重叠与短视价值一致性
3. **可用性闸门决策：** 对于重叠度极低的子域，标记为"RL 受限区"，后续采用高质量子集的行为克隆（BC）对照

### 交付物
- 因果-公平审计报告
- $\pi_{\text{beh}}$ 模型
- $U_{\text{del}}(s,a)$ 实现
- RL/BC 子域清单

### 注意事项
此阶段必须在训练前完成，避免选择偏倚

### 检查点 1
- 数据审计是否发现重大偏见或混淆风险？
- RL可用子域是否足够覆盖目标人群？

## Phase R：双系统奖励设计与定标

**周期：** 2-3 周
**目标：** 设计训练用密集奖励（系统A）和评估用临床指标（系统B），并确保一致性（参考 3.2）

### Task R.1: 系统 B（评估用）指标定义

1. **定义临床终点与过程指标：** 明确终端指标（生存率、ICU天数等）和过程安全指标（药物暴露、AKI代理等）
2. 系统 B 不参与梯度更新

### Task R.2: 系统 A（训练用）奖励塑形

1. **实现奖励函数：**
   $$r_t = w_1\,R_{\text{phys}}(s_t,s_{t+1})\;-\;w_2\,\|a_t-a_{t-1}\|_1\;-\;w_3\,\mathbf{1}[\text{高风险/越界代理}]$$
2. **$R_{\text{phys}}$（生理指标）：** 基于 MAP≥65、乳酸下降等目标。采用连续塑形（Shaping）避免硬阈值抖动
3. **平滑项与风险项：** 惩罚剂量跳变；即时惩罚高危状态
4. **饱和抑制：** 对预测为"终末/不可逆"的状态下调奖励权重，避免无效优化

### Task R.3: 权重标定与一致性检验

1. **权重调优：** 在验证集上搜索最优权重组合 $(w_1, w_2, w_3)$
2. **目标一致性检验（关键）：** 计算验证集上系统 A 累计奖励与系统 B 指标的相关性（Spearman/Pearson）和单调性
3. **回调机制：** 若发现脱钩（A高但B差），必须重新调整权重或 $R_{\text{phys}}$ 设计

### 交付物
- 系统 A 和 B 的实现代码
- 最优权重配置
- 一致性检验报告

### 注意事项
禁止用稀疏的终端生存奖励直接进行训练

### 检查点 2
- 系统A与系统B是否通过一致性检验？
- 若未通过，是否有明确的回调记录和修正方案？

## Phase 0 & 1：基线建立与安全层定义

**周期：** 3-4 周
**目标：** 建立性能基准，定义分层安全架构的L1和L2层

### Task 0.1: 基线复现与 OPE 配置 (Phase 0)

1. **实现基线算法：** BC（特别是在 Phase -1 定义的高质量子集上）、标准 CQL（固定α）
2. **配置 OPE 套件：** 实现 FQE（主用）、WIS、DR
3. 建立评估标准（固定种子、Wilson区间）
4. 确保 OPE 评估器能报告有效样本数（ESS）和权重方差

### Task 1.1: L1 语义安全层（硬约束）构建 (Phase 1)

1. **定义语义目录：** 梳理临床指南与专家知识，明确硬禁忌、剂量边界、绝对不相容条件
2. **实现动作掩码生成器：** 实现函数生成动态可行动作集 $\mathcal{A}_{\text{safe}}(s)$

### Task 1.2: L2 认知安全层（不确定性/OOD）构建 (Phase 1)

1. **风险代理模型训练：** 训练并校准（如等温回归）风险代理模型（预测ΔSOFA恶化、持续低MAP等）
2. **OOD/重叠度检测：** 实现 OOD（分布外）/低重叠度检测器（基于密度估计或集成方法）
3. **阈值设定：** 在验证集上设定 L2 阈值：OOD 阈值 $(\delta)$ 和混淆不确定性 $U_{\text{del}}$ 阈值 $(\kappa)$

### 交付物
- 基线模型性能报告
- OPE 套件
- L1目录与 $\mathcal{A}_{\text{safe}}(s)$ 实现
- L2检测器与阈值

### 注意事项
安全阈值的设定只能基于训练集和验证集

## Phase 2 & 3：核心算法实现、装配与训练

**周期：** 5-7 周
**目标：** 实现核心算法（Filtered-MDP + 自适应保守），装配安全架构，并执行训练（参考 3.3, 3.4）

### Task 2.1: 自适应保守性实现 (Phase 2)

1. **动态 $\alpha(s)$：** 修改 CQL 以支持状态依赖的保守性系数
2. **实现调节逻辑：** $\alpha_{\text{eff}}(s) = \alpha_0 + \gamma\cdot \text{OOD}(s)$（OOD高则更保守）

### Task 2.2: Filtered-MDP 训练实现（关键创新）(Phase 2)

1. **修改 Bellman 备份：** 必须将 L1 动作掩码集成到 Bellman 目标计算中：
   $$y = r + \gamma \max_{a'\in \mathcal{A}_{\text{safe}}(s')} Q_{\theta^-}(s',a')$$
2. **训练/推断同构：** 训练过程中执行动作选择时也必须应用掩码，避免推断期首次覆盖导致的 OOD 失稳

### Task 2.3: 推断期分层裁决器实现 (Phase 2)

1. **实现层级裁决逻辑（L1→L2→L3）：**
   - L1（语义）：检查 $\mathcal{A}_{\text{safe}}(s)$
   - L2（认知）：检查 OOD $\le \delta$ 且 $U_{\text{del}} \le \kappa$
   - L3（RL优化）：执行 $\text{argmax} Q(s,a)$
2. **定义回退策略：** 若候选集为空，执行回退策略（如"行为最近邻的安全动作"或请求人工）
3. **日志记录：** 实现详细的介入日志记录

### Task 3.1: 训练协议与超参优化 (Phase 3)

1. **超参数调优：** 定义搜索空间 $(\alpha_0, \gamma, \delta, \kappa)$。使用验证集进行调优
2. **训练与监控：** 使用多个随机种子训练。使用 OPE（FQE为主）进行早停
3. 监控 ESS 和权重方差确保 OPE 可靠性

### Task 3.2: 混淆悲观化与模型选择 (Phase 3)

1. **混淆悲观化评价：** 在模型选择阶段，使用悲观 Q 值进行评价（不用于训练梯度）：
   $$\tilde Q(s,a)=\hat Q(s,a)-\beta\,U_{\text{del}}(s,a)$$
2. **Pareto 前沿构建：** 选择一组构成"性能-安全-解释" Pareto 前沿的非支配候选策略集（验证H3）

### 交付物
- RF-CQL + Filtered-MDP 完整实现代码
- 推断裁决器
- 训练好的候选策略集

### 注意事项
- 严禁仅在推断期"事后过滤"
- 避免叠加与 CQL 冲突的机制（如 CMDP 拉格朗日项）

### 检查点 3
- Filtered-MDP是否正确实现（掩码进入Bellman备份）？
- 训练与推断是否同构？
- OPE评估是否稳定（ESS足够高）？

## Phase 4：主评估：性能—安全—解释—公平 四合一

**周期：** 4-5 周
**目标：** 在隔离的测试集上全面评估策略，验证假设 H1-H4（参考四）

### 执行步骤（解封测试集并执行）

#### 1. 有效性（性能）评估 (H1)
- 报告总体及高危人群（SOFA≥11）的生存率/回报（系统B指标）
- **统计报告（参考七）：** 提供 95% Wilson 置信区间、显著性检验、效应量
- 执行多重比较控制（BH-FDR）

#### 2. 安全性评估 (H1)
- **硬安全：** 硬禁忌违例率（验收红线：必须≈0）
- **软安全：** 风险违例率和 OOD 行为率（应显著低于基线）
- **可操作性：** L1/L2 过滤器介入率与告警负担分析

#### 3. 可审计性（解释）评估 (H2)
- 计算 LEG/SHAP Top-k 特征（关注MAP、乳酸等）的稳定性（Jaccard/RankCorr）
- 进行单调性核验和临床一致性评估
- 生成可供审计的"策略卡"
- 执行关键病例复盘（Case Review）

#### 4. 公平性评估 (H4)
- 在关键分层中评估性能、安全指标和过滤器介入率是否存在系统性不公差异

#### 5. 伪动态/影子近似评估
- **A档（必做）：** 流式回放测试集，记录策略与医生分歧率、时间稳定性、OOD频次和过滤器介入率

### 交付物
- 最终综合评估报告
- 统计分析结果
- 策略卡
- 病例复盘文档

### 注意事项
- 必须报告 SOFA≥11 分层结果
- 避免仅依赖总体均值

### 检查点 4（项目验收）
- 策略是否满足所有验收标准？
- 特别是安全红线（硬禁忌违例≈0）和公平门槛是否达标？

## 关键原则与铁律

### 数据管理铁律
1. **测试集封存**：Phase 0 后封存，Phase 4 才解封
2. **验证集用途**：仅用于超参调优和阈值设定
3. **训练集专用**：仅用于模型训练和预处理器拟合

### 安全优先原则
1. **硬约束零容忍**：硬禁忌违例必须≈0
2. **训练推断同构**：Filtered-MDP 必须在训练时就集成掩码
3. **事后过滤禁止**：不允许仅在推断期过滤

### 可复现性要求
1. **固定随机种子**：所有实验必须可复现
2. **统一精度**：全局使用 FP32
3. **版本控制**：代码、配置、数据预处理全部版本化
4. **实验追踪**：使用 MLflow 等工具记录所有实验

### 评估严格性
1. **分层报告**：必须报告高危人群（SOFA≥11）结果
2. **置信区间**：所有指标提供 95% Wilson 区间
3. **多重校正**：执行 BH-FDR 控制
4. **公平性审查**：按关键群体分层评估
